Sobre mí

Soy católica y valoro la fe como una guía en la vida diaria. Busco vivir de acuerdo con principios como el amor, la esperanza y el servicio a los demás, encontrando en Jesús un ejemplo a seguir y una fuente de inspiración.

¿Qué es la Semana Santa?

Es la celebración más importante del cristianismo/catolicismo, donde se conmemora la pasión, muerte y resurreción de Jesucristo.

Días importantes de Semana Santa

  • Domingo de Ramos: Es cuando Jesús llega a Jerusalén.
  • Jueves Santo: Se celebra la Última Cena.
  • Viernes Santo: Se conmemora la crucifixión y muerte de Jesús.
  • Sábado Santo: Día de silencio y espera.
  • Domingo de Resurreción: Se celebra que Jesús resucitó.

Preguntas del Laboratorio #1

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet: Red global de computadoras interconectadas World Wide Web: Sistema de documentos e hipervínculos que funcionan sobre Internet.

¿Cuáles son las partes de un URL?

  • Protocolo https://
  • Dominio (nombre del servicio)
  • Puerto (Canal de comunicación)
  • Ruta (Endpoint solicitado)
  • Query string (Parámetros adicionales)
  • Fragmento (No llega al servidor)

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Obtiene un recurso.
  • HEAD: Verifica si un recurso existe sin descargarlo.
  • POST: Crea un recurso nuevo.
  • PUT: Reemplaza un recurso completo.
  • PATCH: Actualiza parcialmente.
  • DELETE: Elimina un recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Método POST porque permite enviar los datos en el cuerpo de la solicitud, evitando que información sensible como usuario y contraseña sea visible en la URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Método GET, permite solicitar y obtener recursos del servicio a través de una URL, sin modificar la información existente.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No hay ningún error, el código 200 significa que la petición fue exitosa, y el recurso se devuelve en el cuerpo.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, ya que el error 404 significa que el recurso no fue encontrado, lo cual puede deberse a enlaces rotos, URLs mal configurados o contenido eliminado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, ya que el error 500 indica un problema interno en el servidor, generalmente causado por fallas en el código, configuraciones incorrectas o errores en la lógica.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que este atributo ya no se recomienda usar, porque ha sido reemplazado por mejores prácticas o tecnologías.

  • font
  • center
  • big
  • striker

¿Qué componentes de estructura y estilo tiene una tabla?

Estructura:

  • table: define la tabla
  • tr: fila
  • td: celda de datos
  • th: celda de encabezado
  • thead: encabezado de la tabla
  • tbody: cuerpo
  • tfoot: pie de tabla

Estilo (principalmente con CSS):

  • Bordes (border)
  • Espaciado (padding, margin)
  • Alineación (text-align)
  • Colores (background-color, color)
  • Tamaño (width, height)

¿Cuáles son los principales controlesde una forma HTML5?

  • input (con tipos como: text, password, email, number, date, checkbox, radio, file)
  • textarea (campo de texto largo)
  • select (lista desplegable)
  • option (opciones dentro de select)
  • button (botón)
  • label (etiqueta descriptiva)
  • fieldset y legend (agrupación de campos)

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

Ciclo de vida de los sistemas de información: Es el conjunto de etapas por las que pasa un sistema desde que se concibe hasta que deja de usarse:

  • Planificación
  • Análisis
  • Diseño
  • Desarrollo
  • Implementación
  • Mantenimiento
  • Retiro del sistema

Ciclo de desarrollo de sistemas de información (SDLC): Es el proceso estructurado para crear un sistema:

  • Recolección de requerimientos
  • Análisis del sistema
  • Diseño
  • Programación (codificación)
  • Pruebas
  • Implementación
  • Mantenimiento
