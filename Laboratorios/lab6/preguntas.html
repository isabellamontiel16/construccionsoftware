Cuestionario — CSS Avanzado y Frameworks

1. ¿Qué ventajas ofrecen las variables CSS sobre los valores hardcodeados?

Permiten reutilizar valores, facilitan el mantenimiento, aseguran consistencia y permiten crear temas dinámicos como modo oscuro.

2. ¿En qué casos conviene declarar variables fuera de :root?

Cuando se quieren aplicar solo a un componente o contexto específico, como un tema oscuro o una sección particular.

3. ¿Cómo se usan las variables CSS para modo oscuro?

Se definen valores en :root y se sobrescriben en una clase como body.dark para cambiar todo el tema dinámicamente.

4. ¿Qué sucede si usas una variable no declarada?

La propiedad no se aplica, o se puede usar un valor por defecto con var(--variable, fallback).

5. ¿Diferencia entre mobile-first y desktop-first?

Mobile-first diseña primero para móvil y escala con min-width. Desktop-first comienza en escritorio y usa max-width.

6. ¿Breakpoints más comunes y por qué?

480px, 768px, 1024px y 1280px porque corresponden a tamaños reales de dispositivos.

7. ¿Cómo afecta el viewport meta tag?

Permite que la página se adapte correctamente al tamaño del dispositivo y evita escalados incorrectos en móviles.

8. Técnicas además de media queries

Flexbox, Grid, unidades relativas (rem, %, vw), clamp(), max-width e imágenes responsivas.

9. ¿Cómo identificar qué regla CSS se aplica?

En DevTools, la regla activa no está tachada. Gana la más específica o la última declarada.

10. ¿Qué muestra la pestaña Computed?

El valor final de cada propiedad después de aplicar herencia, conflictos y variables.

11. ¿Cómo simular dispositivos?

Usando DevTools con Ctrl + Shift + M para probar diferentes resoluciones y dispositivos.

12. ¿Cuándo usar Bootstrap vs Tailwind?

Bootstrap para prototipos rápidos. Tailwind para diseños personalizados.

13. ¿Qué significa utility-first?

Que las clases hacen una sola cosa, como padding, color o tamaño.

14. ¿Cómo afecta al tamaño del bundle?

Bootstrap puede ser pesado. Tailwind puede optimizarse pero también crecer si no se configura.

15. ¿Se pueden combinar frameworks?

Sí, pero no es recomendable por conflictos, duplicación y dificultad de mantenimiento.
